# PTC Creation Checklist

Use this checklist to ensure your Procedure Test Case is complete and high-quality.

## Phase 1: Planning ✓

- [ ] Procedure to be tested is clearly identified
- [ ] Test objectives are defined
- [ ] Stakeholders are identified
- [ ] Priority level assigned
- [ ] Test type determined (Functional/Integration/etc.)

## Phase 2: Preparation ✓

- [ ] Unique test case ID assigned (PTC-YYYY-NNN)
- [ ] Template copied and renamed
- [ ] Test case information section completed
- [ ] Procedure details documented
- [ ] Prerequisites identified and listed
- [ ] Environment requirements documented
- [ ] Access requirements documented
- [ ] Test data requirements identified
- [ ] Test data prepared or sourced

## Phase 3: Test Design ✓

- [ ] Test steps clearly defined
- [ ] Each step has specific action
- [ ] Input data specified for each step
- [ ] Expected results are specific and measurable
- [ ] Steps are numbered sequentially
- [ ] Steps cover the complete procedure
- [ ] Both positive and negative scenarios included
- [ ] Edge cases considered
- [ ] Pass/fail criteria clearly defined
- [ ] Acceptance criteria documented

## Phase 4: Review ✓

- [ ] Test case reviewed for clarity
- [ ] Prerequisites are complete and accurate
- [ ] Test steps are reproducible
- [ ] Expected results are realistic
- [ ] Test data is appropriate
- [ ] No assumptions undocumented
- [ ] Language is clear and unambiguous
- [ ] Peer review completed
- [ ] Feedback incorporated

## Phase 5: Approval ✓

- [ ] Test case submitted for review
- [ ] Review comments addressed
- [ ] Final approval obtained
- [ ] Status updated to "Approved"

## Phase 6: Execution ✓

- [ ] Test environment prepared
- [ ] Prerequisites verified
- [ ] Test data available
- [ ] Execution started with proper logging
- [ ] Each step executed in sequence
- [ ] Actual results recorded for each step
- [ ] Screenshots/evidence captured
- [ ] Status updated for each step (Pass/Fail/Blocked)
- [ ] Defects documented as found
- [ ] Execution timestamp recorded

## Phase 7: Documentation ✓

- [ ] Actual results summary completed
- [ ] All fields in template filled
- [ ] Defects logged in tracking system
- [ ] Defect IDs added to PTC
- [ ] Test evidence attached/referenced
- [ ] Execution details section completed
- [ ] Overall test result determined
- [ ] Conclusion section completed

## Phase 8: Post-Execution ✓

- [ ] Results reviewed with stakeholders
- [ ] Defects assigned and prioritized
- [ ] Recommendations documented
- [ ] Follow-up actions identified
- [ ] Test metrics recorded
- [ ] PTC status updated to "Executed"
- [ ] Files properly saved and backed up

## Phase 9: Reporting ✓

- [ ] Test summary report created
- [ ] Results shared with stakeholders
- [ ] Metrics updated in dashboard
- [ ] Lessons learned documented
- [ ] Repository updated

## Phase 10: Archival ✓

- [ ] PTC saved in designated location
- [ ] Naming convention followed
- [ ] Version control updated
- [ ] Test evidence archived
- [ ] Documentation indexed
- [ ] Related documents linked

## Quality Gates

### Before Execution
- [ ] All Phase 1-5 items complete
- [ ] Approval obtained
- [ ] Resources available

### Before Completion
- [ ] All steps executed
- [ ] All results documented
- [ ] All defects logged

### Before Archival
- [ ] All documentation complete
- [ ] All approvals obtained
- [ ] All evidence preserved

## Common Issues Checklist

Verify you haven't made these mistakes:

- [ ] No vague expected results like "system works"
- [ ] No missing prerequisite information
- [ ] No undocumented test data
- [ ] No skipped steps in documentation
- [ ] No uncaptured evidence
- [ ] No unlogged defects
- [ ] No missing timestamps
- [ ] No duplicate test case IDs

## Final Validation

- [ ] Test case is complete
- [ ] Test case is accurate
- [ ] Test case is reproducible
- [ ] Test case is maintainable
- [ ] Test case follows standards
- [ ] Test case adds value
